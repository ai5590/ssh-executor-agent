Создай production-ready Java 17 проект с использованием Spring Boot и Gradle.

Название проекта: ssh-executor-agent

Назначение:
HTTP сервис для выполнения команд по SSH на удалённых серверах.

Требования:

========================================
ТЕХНОЛОГИИ
========================================

- Java 17
- Gradle (Groovy DSL)
- Spring Boot (web)
- Jackson
- Apache Mina SSHD client
- Без лишних зависимостей

========================================
ФУНКЦИОНАЛ
========================================

1) Endpoint #1

POST /exec

Request JSON:
{
  "server": "core01",
  "command": "uname -a"
}

Response JSON:
{
  "result": "<одна строка с результатом выполнения>"
}

Поведение:
- Загружать servers.json при старте
- Если server не найден → вернуть JSON с ошибкой
- Если command пустая → вернуть JSON с ошибкой
- Поддержка SSH:
  - авторизация по ключу
  - авторизация по паролю
- Таймаут выполнения команды: 30 секунд
- Лимит stdout + stderr: максимум 10 KB
- Если превышен лимит — обрезать и добавить "[OUTPUT TRUNCATED]"

Формат result (одна строка, с \n):

=== EXECUTION RESULT ===
server: core01
exit_code: 0
duration_ms: 142

--- STDOUT ---
...

--- STDERR ---
...

=== END ===

========================================

2) Endpoint #2

GET /servers

Возвращает:

{
  "result": "core01 192.168.1.250\nvps01 10.10.10.2"
}

То есть:
- Только имя сервера и IP
- В одной строке
- Через \n

========================================

3) servers.json

Файл в корне проекта:

{
  "core01": {
    "host": "192.168.1.250",
    "user": "root",
    "port": 22,
    "authType": "key",
    "keyPath": "/keys/core01_ed25519"
  },
  "vps01": {
    "host": "10.10.10.2",
    "user": "ubuntu",
    "port": 22,
    "authType": "password",
    "password": "CHANGE_ME"
  }
}

Логика:
- authType = "key" → использовать приватный ключ
- authType = "password" → использовать пароль

    ExecController
